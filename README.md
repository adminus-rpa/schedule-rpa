# Веб-расписание — production build (v3)

## 🚀 Установка зависимостей и запуск проекта

### 1. Системные зависимости (ОБЯЗАТЕЛЬНО для macOS/Linux)
Бэкенд читает базу данных Microsoft Access (`.mdb`), поэтому на Mac/Linux необходимо установить системную утилиту `mdbtools`. Без неё сервер выдаст ошибку `mdbtools не установлен`.
- **На macOS**: `brew install mdbtools`
- **На Ubuntu/Debian**: `sudo apt-get install mdbtools`

### 2. Запуск Бэкенда (Python)
В корне проекта есть удобный скрипт `run.sh`, который автоматически создаст виртуальное окружение `.venv`, установит библиотеки из `requirements.txt` и запустит сервер.
```bash
# Находясь в корне проекта
./run.sh
```
*Бэкенд будет запущен на http://localhost:9090*

### 3. Запуск Фронтенда (Node.js + Svelte 5 + Vite)
Фронтенд-приложение лежит в папке `frontend`. Вам потребуется установленный Node.js (желательно v20+).
```bash
# Откройте новую вкладку терминала и перейдите в папку frontend
cd frontend

# Установите зависимости (нужно выполнить только один раз)
npm install

# Запустите режим разработчика
npm run dev
```
*Фронтенд будет запущен на http://localhost:5173 и будет автоматически проксировать API-запросы к бэкенду.*

# Веб-расписание — production build (v2)

Веб-приложение для трансляции расписания занятий на телевизорах-терминалах.
Читает базу Microsoft Access («Экспресс-расписание ВУЗ») и показывает
расписание в виде **«островков групп»**: каждый остров = одна группа,
внутри — карточки-пары, соединённые пунктирной линией только в зазорах
между ними.

## 🆕 Обновления v2

- **Дизайн «островков»** — вертикальные группы-картелы, карточки-пары с
  отступами и пунктирными коннекторами (линия НЕ заходит на сами карточки).
- **Левая колонка всегда показывает 7 пар**, всё вмещается в экран без скролла.
- **Пастельно-сиреневая подсветка** текущей пары (`#b39ddb`, лёгкая, прозрачная).
- **Прогресс-бар пагинации** сверху панелей вместо цветной линии.
- **F11 / кнопка в шапке** — полноэкранный режим с автоскрытием курсора.
- **«КОЛЛЕДЖ» / «ВЫСШЕЕ ОБРАЗОВАНИЕ» капсом** + анимация fly-in/fly-out
  при переключении панелей в режиме alternate.
- **Автозамены НА СЕРВЕРЕ** — любые варианты ЭИОС («с пр ЭИОС», «лек. ЭИОС»,
  «пр. ЭИОС», «Э.И.О.С», «Электронная информационно-образовательная среда»...) →
  единый вид «ЭИОС».
- **Расширенные анимации**: `swap` (дефолт), `slide`, `fade`, `zoom`, `flip`,
  `cube`, `stack`, `reveal`, `none`. Сменяется ключом `display.page_animation`.
- **Рассеянное свечение островков** (по умолчанию выключено) —
  `display.island_glow_enabled: true` + `island_glow_radius`, `island_glow_opacity`,
  `island_glow_pulse` в конфиге.
- **Отаказоустойчивость»**: SSE-reconnect с экспоненциальным backoff, global
  error-boundary, ghost-sweeper (авточистка «зависших» страниц после сбоев
  анимации), safe-timer обёртки.

### Ключевые баги исправлены

- ¡ Серые ячейки-призраки после смены экрана (агрессивная чистка старых `.page` в
  `animateSwap()`, `cleanupPanel()` перед fullRender и ghost-sweeper раз в 4 с).
- ¡ Подсветка текущей пары стабильна (обновляется каждые 30 сек. + при возврате
  вкладки из фона).
- ¡ Соединительная линия НЕ заходит на карточки (отдельный DOM-элемент
  `.lesson-connector` между карточками).

### Полноэкранный режим

- **F11** — включить/выключить fullscreen.
- **Кнопка в правой части шапки** — тот же эффект, иконка меняется автоматически
  на «выйти».
- **`behavior.fullscreen_on_first_interaction: true`** — первый клик/клавиша
  автоматически входит в fullscreen (типично для терминалов).

## ✨ Основные возможности

- 🗂 **Вертикальные колонки групп** — одна колонка = вся дневная сетка одной
  группы. Все линии-разделители между парами строго совпадают по всей ширине.
- 📄 **Пагинация групп** — если не помещаются на экран, показываются страницами
  с плавной анимацией (slide / fade / zoom / flip / none, настраивается в конфиге).
- 🔁 **Alternate-режим** — сначала показываются ВСЕ страницы групп одного
  подразделения, потом — другого. И так по кругу.
- ↕️ **Автоскролл** внутри страницы — если в группе больше пар, чем помещается
  по высоте, столбцы плавно прокручиваются вниз-вверх (скорость и пауза
  настраиваются в конфиге).
- 🏛 **Кабинет фиксированной ширины «под 4 символа»** — цифры не пляшут,
  окончания вроде `КЮ`, `зал`, `Ст.с` автоматически переносятся на вторую строку.
- 🎨 **Цвет номера кабинета зависит от этажа** (первая цифра номера = этаж).
  Настраивается через `theme.floor_colors`.
- 🔤 **Обрезка длинного названия предмета** до одной строки с троеточием (CSS
  ellipsis + опциональный жёсткий лимит символов).
- ✂️ **Спец-замены** в названиях предметов и кабинетов — regex-словарь в конфиге.
  Пример: любое написание «ЭИОС» / «электронная информационно-образовательная
  среда» приводится к «ЭИОС».
- 🌓 **Тема light по умолчанию**, есть dark / uni_blue / warm / contrast.
- 🌆 **Фоновая картинка** через `theme.background_image` — путь к файлу в
  `/static/`, полный URL, либо `""` для отключения. Меняется на лету.
- 🕒 **Автосмена дня в полночь** — если `period: "day"`, страница сама
  переключается на новый день.
- 🟠 **Подсветка текущей пары** — только по времени (без сравнения даты),
  чтобы в предпросмотре другого дня подсветка не мешала.
- 🔑 **Клавиша `M`** — показать/скрыть верхнюю панель. Работает и при
  русской, и при английской раскладке.
- 📅 **Клик по дате** в шапке открывает системный календарь.
- ⌨️ **Русская и английская раскладки** — все шорткаты (`M`, `T`, `R`, стрелки)
  работают на обеих раскладках (используется `event.code`).
- 🔴 **Push-обновления через SSE** + fallback на polling, если SSE недоступен.
- 🩺 **Отказоустойчивость** — кэш снапшотов, автоматический переподключ к БД,
  индикатор статуса (`● онлайн` / `● БД: кэш` / `● нет данных`).
- 🖼 **Локальные шрифты** — Inter (400/500/600/700/800) в `/static/fonts/`,
  SVG-иконки в `/static/img/icons.svg`. Всё работает без интернета.

## ⌨ Клавиатурные шорткаты

| Клавиша | Действие |
|---|---|
| `←` / `→` | Предыдущий / следующий период |
| `T` (или `Е`) | Перейти к сегодняшней дате |
| `M` (или `Ь`) | Показать / скрыть верхнее меню |
| `R` (или `К`) | Принудительно обновить данные |
| `Esc` | Показать верхнее меню (если было скрыто) |

Клавиши работают одинаково при русской и английской раскладках клавиатуры.

## ⚙️ Конфигурация (config/config.yaml)

Изменения применяются **на лету** — сервер перечитывает файл раз в 5 сек,
клиенты получают push-обновление через SSE.

### Раздел `display`

| Параметр | Что делает |
|----------|------------|
| `period` | Длительность: `day` / `week` / `2weeks` / `3weeks` / `month` |
| `start` | С какого дня отсчёт: `today` / `monday` |
| `show_weekends` | Показывать субботу/воскресенье |
| `hide_empty_groups` | Не показывать группы без занятий в периоде |
| `layout` | `left_college` / `left_university` / `vertical` / `college_only` / `university_only` / **`alternate`** (по ТЗ) |
| `alternate_interval_seconds` | Раньше — интервал переключения панелей. Теперь панель переключается только когда прошли ВСЕ страницы групп (см. `page_interval_seconds`) |
| `show_teacher` | `true` / `false`. Если `false` — блок преподавателя полностью исчезает из карточки |
| `groups_per_page` | Сколько групп в странице (0 = авто, по ширине экрана) |
| `group_col_min_px` | Минимальная ширина колонки группы (для авто-расчёта) |
| `group_col_max_px` | Максимальная ширина колонки |
| `page_interval_seconds` | Интервал переключения страниц групп (сек) |
| `page_animation` | Тип анимации: `slide` / `fade` / `zoom` / `flip` / `none` |
| `page_animation_ms` | Длительность анимации переключения (мс) |
| `auto_scroll` | Включён ли автоскролл (true/false) |
| `auto_scroll_speed` | Скорость (пикселей в секунду) |
| `auto_scroll_pause_ms` | Пауза в начале и конце цикла (мс) |
| `subject_max_chars` | Жёсткий лимит длины предмета (0 = только CSS ellipsis) |
| `subject_replacements` | Regex-словарь для автозамен в предметах |
| `room_replacements` | Regex-словарь для автозамен в кабинетах |

### Раздел `theme`

| Параметр | Что делает |
|----------|------------|
| `preset` | `light` (по умолчанию) / `dark` / `uni_blue` / `warm` / `contrast` |
| `overrides` | Точечное переопределение любого цвета |
| `background_image` | URL, файл в `/static/`, или пусто |
| `background_opacity` | 0.0–1.0 |
| `floor_colors` | Цвета кабинета по этажу — ключи `"1"`..`"9"` + `default` |

### Раздел `typography`

| Параметр | Что делает |
|----------|------------|
| `cell_font_size` | Размер шрифта в карточке (rem) |
| `header_font_size` | Размер заголовков панелей (rem) |
| `title_font_size` | Размер основного заголовка (rem) |
| `font_family` | Цепочка шрифтов (по умолчанию Inter локально) |

### Раздел `behavior`

| Параметр | Что делает |
|----------|------------|
| `hide_topbar_on_load` | Скрывать верхнее меню при загрузке |
| `client_refresh_seconds` | Интервал polling'а (fallback) |
| `client_config_poll_seconds` | Интервал опроса конфига клиентом |
| `show_status_indicator` | Показывать `● онлайн / кэш / ошибка` в шапке |
| `show_clock` | Показывать текущее время в шапке |

## 🔌 REST API + SSE

| Endpoint | Метод | Назначение |
|---|---|---|
| `/` | GET | HTML-страница расписания |
| `/api/schedule?start=YYYY-MM-DD&period=day` | GET | JSON-снимок расписания на диапазон |
| `/api/schedule?start=YYYY-MM-DD&end=YYYY-MM-DD` | GET | JSON-снимок на произвольный диапазон |
| `/api/config` | GET | Тема, поведение, шрифты, звонки |
| `/api/status` | GET | Состояние БД и версии |
| `/api/events` | GET | **SSE-поток** — push-обновления |
| `/api/sync` | POST | Форс-синхронизация (только с localhost) |
| `/healthz` | GET | Health-check для мониторинга |

**SSE-события**:
- `hello` — соединение установлено
- `data_updated` — данные пересинхронизированы, клиент подхватит `/api/schedule`
- `config_updated` — конфиг перечитан, клиент подхватит `/api/config`
- `sync_error` — не удалось прочитать БД (данные из кэша)
- `ping` — heartbeat раз в 25 сек

## 🩺 Отказоустойчивость

- Соединение с БД переустанавливается при ошибке
- Кэш снапшотов — если БД временно недоступна, отдаётся последний удачный
- В интерфейсе индикатор: `● онлайн` / `● БД: кэш` / `● нет данных`
- Если SSE-канал упал → клиент автоматически переходит на polling и пытается
  переподключить SSE каждые 30 сек
- Ротируемые логи (5 файлов по 5 МБ)
- Автоматическая смена дня в полночь (таймер + guard раз в минуту)

## 🛠 Устранение неполадок

**«Не подключается к БД» на Windows:**
- Проверьте разрядность Access Database Engine и Python
  (`python -c "import struct; print(struct.calcsize('P')*8)"`)
- Проверьте права доступа под пользователем службы к UNC-пути

**«SSE не подключается» (индикатор `polling`):**
- Проверьте, что перед приложением нет proxy, который буферизует ответы
  (для nginx: `proxy_buffering off; proxy_http_version 1.1; proxy_read_timeout 3600s;`)
- Firewall не режет long-poll соединения

**Слишком мелко/крупно:**
- `typography.cell_font_size: 1.2` (или 0.85) — обновление за 5 сек без перезагрузки

**Мало групп на странице:**
- Уменьшите `display.group_col_min_px` (например до 180) — будет больше колонок
- Или задайте `display.groups_per_page: 8` — фикс. число

**Автоскролл слишком быстрый / медленный:**
- `display.auto_scroll_speed: 40` (пикселей в секунду)
- `display.auto_scroll_pause_ms: 3000` (пауза в конце цикла)

**Хочу отключить автоскролл:**
- `display.auto_scroll: false`
